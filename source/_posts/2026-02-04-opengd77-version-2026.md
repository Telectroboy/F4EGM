### Message par Anonyme, 2026-02-04

Du coup, j'ai poussé l'analyse beaucoup plus loin et j'ai poussé un GitHub avec un changelog => [https://github.com/Telectroboy/opengd77](https://github.com/Telectroboy/opengd77) Nouvelles options / menus (2026)
APRS
APRS Power : choix « From Master » ou niveau de puissance dédié pour le beaconing APRS.
General options
Channels are read only : liste des canaux en lecture seule (pas d’édition depuis le radio). Bit : BIT_CHANNELS_ARE_READ_ONLY.
Display options (MD9600)
Time in header : affichage de l’heure dans l’en-tête (BIT_DISPLAY_TIME_IN_HEADER).
DMR Last talker on screen : durée d’affichage du dernier parleur (0–30 s, lastTalkerOnScreenTimer).
TX Inhibit
Option globale TX Inhibit (BIT_TX_INHIBIT).
Notifications associées : NOTIFICATION_TYPE_BEARING, NOTIFICATION_ID_BEARING, NOTIFICATION_ID_TX_INHIBIT.
UI
UI uses double height : texte en double hauteur (BIT_UI_USES_DOUBLE_HEIGHT).
GPS
Mode et débit en un seul champ gpsModeAndBaudsIndex (4 bits mode, 4 bits index bauds). Macros SETTINGS_GPS_MODE_GET/SET et SETTINGS_GPS_BAUDRATE_GET/SET.
Roaming
Nouveau type roaming_t (Off, Manual, 5 km, 10 km, 20 km) et champ roaming dans les paramètres.
Nouvelles fonctions / API
settingsLocationGetLatitude(), settingsLocationGetLongitude(), settingsLocationIsValid().
distanceBetweenTwoCoords(lat1, lon1, lat2, lon2, inKm).
buildMaidenHead().
Paramètres : bitfieldOptions devient un tableau par « banque » pour de futurs bits ; nouveaux bits listés ci-dessus.
Langues
Version du struct de langue passée à 0x05.
Nouvelles chaînes : tx_inhibit, roaming, et celles des nouvelles options (ex. « Ch Power », « From Master » pour APRS). Bugs corrigés
Conversion latitude/longitude (fixed-point 24/32 bits)
En 2024, les fonctions latLongDoubleToFixed24 / latLongDoubleToFixed32 utilisaient de mauvais décalages de bits (ex. << 23 au lieu de << 15 pour le format 24 bits). En 2026, les formules sont corrigées et les noms harmonisés en latLon*.
Impact : position GPS, APRS, Maidenhead, tri par distance plus corrects.
Format et version APRS
Nouvelle version du format APRS (APRS_MAGIC_VERSION 0x4152 → 0x4153), initialisation explicite de APRS_Buf->magicVer et type renommé en CodeplugCustomDataType_t pour éviter des usages incohérents.
Gestion de la position (robustesse)
Position regroupée dans nonVolatileSettings.location.lat/.lon et accès via settingsLocationGetLatitude(), settingsLocationGetLongitude() et settingsLocationIsValid().
Impact : une seule source de vérité pour la position, moins de risques d’incohérence ou d’accès directs incorrects.
Améliorations (sans bug précis identifié)
Hotspot : displayFWVersion dans une structure dédiée + support optionnel de l’affichage en double hauteur.
Distance : nouvelle fonction distanceBetweenTwoCoords() et utilisation de l’API de position pour distanceToLocation().
Maidenhead : fonction buildMaidenHead() avec les bons multiplicateurs et usage de location.lat/.lon.
Pile logicielle : mises à jour FreeRTOS, HAL STM32, USB, CMSIS (dont TrustZone).
Codeplug : renommage des types (struct_* → Codeplug*) pour plus de cohérence.
En pratique, les changements les plus importants entre 2024 et 2026 sont la correction des conversions de position (bug pouvant fausser GPS/APRS/affichage), l’API centralisée pour la position, et les mises à jour de la pile (FreeRTOS, HAL, USB). Le détail est dans CHANGELOG_BUGS_ET_AMELIORATIONS_2024_vers_2026.md.
